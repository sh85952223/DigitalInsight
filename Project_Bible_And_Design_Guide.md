# Digital Insight Universe: Project Bible & Design Guide

> **문서 버전**: 1.0  
> **최종 업데이트**: 2026-01-15  
> **목적**: 'Digital Insight' 시리즈의 세계관, 디자인, 경험(UX)를 정의하여 향후 프로젝트(2탄, 확장판)의 일관성을 유지하고 기획의 기준점으로 삼음.

---

## 1. 전체 테마 정의 (The Core Theme)

### 1.1 핵심 세계관: "The Hearth & The Glitch"
이 프로젝트는 단순한 교육용 앱이 아닌, **비밀 요원 양성 시뮬레이션**입니다. 사용자는 UI/UX라는 '디지털 통찰력(Digital Insight)'을 무기로 디지털 세상을 분석하고 수호하는 요원이 됩니다.

*   **배경 (Setting)**: 겉보기엔 평범한 디지털 세상이지만, 그 이면에는 소스 코드와 디자인 원칙으로 이루어진 '매트릭스' 같은 구조가 존재합니다.
*   **사용자 페르소나 (User Persona)**: 훈련생(Trainee) -> 정식 요원(Agent).
*   **조력자 (The Mentor)**: '교관' 또는 '디지털 가이드'. 차갑지만 유능하며, 때로는 시공간을 넘나드는(중세시대 점원 등) 모습으로 나타나 깨달음을 줍니다.
*   **핵심 갈등 (Conflict)**: '혼란스러운 사용자 경험(Bad UX)'과 '깨진 인터페이스(Glitch/Bug)'가 주된 적입니다. 이를 질서(Good UI/UX)로 바로잡는 것이 목표입니다.

### 1.2 감정 키워드 & 분위기
*   **Primary**: `Cyberpunk`, `Professional`, `Secretive`, `High-Tech`
*   **Secondary**: `Immersive`, `Mysterious`, `Glitchy` (불안정함 속의 질서)
*   **Keywords**:
    *   **해킹(Hacking)**: 지루한 학습이 아닌, 시스템을 뚫고 들어가는 능동적 행위.
    *   **각성(Awakening)**: 모르던 것을 알게 되는 순간 시각적/청각적 쾌감 제공.
    *   **전이(Transition)**: 시공간(중세 ↔ 미래)을 넘나드는 극적인 전환.

### 1.3 2탄 확장 방향
*   **1탄의 유산**: 기본 소양(UI/UX 기초)을 갖춘 요원의 탄생.
*   **2탄의 방향**: "현장 투입". 시뮬레이션을 넘어 실제 ‘오염된 웹사이트’나 ‘망가진 앱’세계로 침투합니다. 더 강력한 바이러스(다크 패턴)와 싸우는 실전 미션 위주로 전개될 것입니다.

## 2. 페이지 구조 및 진행 흐름 (Narrative UX Flow)

모든 프로젝트는 **"초대 → 훈련(몰입) → 검증 → 보상"**의 4단계 구조를 따릅니다.

### 2.1 단계별 구성
1.  **Onboarding (초대 & 접속)**
    *   **역할**: 현실 세계와 단절하고 디지털 세계로 접속하는 느낌 부여.
    *   **연출**: 터미널 부팅, 보안 검사, ID 카드 발급, 글리치 효과.
    *   **UX**: 사용자의 호기심을 자극하여 자발적으로 'Start'를 누르게 유도.

2.  **Immersion (시나리오 전개)**
    *   **역할**: 학습 주제(UI/UX)를 스토리 속에 자연스럽게 녹임.
    *   **연출**: 예상치 못한 환경(예: 중세 카페)으로의 전이. *낯선 환경에서 익숙한 개념(UI/UX)을 재발견하게 함.*
    *   **UX**: 탐색형 인터랙션 (클릭하여 단서 찾기, 대화하기).

3.  **Action/Quiz (핵심 이벤트)**
    *   **역할**: 배운 내용을 검증. 지루한 객관식 따위가 아닌 '미션 수행' 형태.
    *   **연출**: 긴급 상황 발생(사이렌, 붉은색 경고), 제한 시간, 긴장감 있는 BGM.
    *   **UX**: 드래그 앤 드롭, 패턴 맞추기 등 직관적인 조작.

4.  **Reward (마무리 & 보상)**
    *   **역할**: 성취감 고취 및 요원으로서의 소속감 부여.
    *   **연출**: 자격증(Certificate) 발급, 화려한 도장 꽝, 팡파레/박수 소리.
    *   **UX**: 결과물 공유 또는 '본부 복귀' 버튼을 통한 깔끔한 종료.

## 3. 디자인 시스템 (Design System)

### 3.1 색상 팔레트 (Cyber-Agent Color Scheme)
기본적으로 **Dark Mode**를 지향하며, 빛(Neon)을 통해 정보를 전달합니다.

*   **Backgrounds**:
    *   `Pure Black (#000000)`: 무한한 디지털 공간. (가장 기본)
    *   `Zinc-950 (#09090b)` ~ `Zinc-900`: UI 컨테이너, 카드 배경.
*   **Primary (Brand)**:
    *   `Cyan-400 (#22d3ee)` ~ `Cyan-500`: 긍정, 시스템 정상, 데이터, 홀로그램.
    *   `Green-500 (#22c55e)`: 성공, 해킹 완료, 승인(Approved).
*   **Secondary (Alert/Emphasis)**:
    *   `Red-600 (#dc2626)`: 경고, 에러, 금지, 긴급 상황.
    *   `Amber-500 (#f59e0b)`: 주의, 중요 데이터 강조.
*   **Effect Colors**:
    *   `White (Text)`: 높은 가독성.
    *   `Gray-400 ~ 600`: 비활성, 보조 텍스트 (단, 너무 어둡지 않게).

### 3.2 타이포그래피 (Typography)
폰트는 세계관의 성격을 규정하는 가장 중요한 요소입니다.

1.  **Display / Heading (미래적, 기술적)**
    *   **Font**: *Orbitron, Russo One* (없을 경우 굵은 고딕체에 영문은 기하학적 폰트).
    *   **Usage**: 큰 제목, 시스템 메시지, 'SYSTEM READY' 같은 HUD 요소.
    *   **Style**: 대문자 위주, 자간(Letter-spacing) 넓게(`tracking-widest`).

2.  **Body / Dialogue (가독성, 현대적)**
    *   **Font**: *Noto Sans KR, Pretendard, Inter*.
    *   **Usage**: 대화창, 본문 설명, 긴 텍스트.
    *   **Style**: 깔끔하고 중립적인 산세리프. `leading-relaxed` (줄간격 여유).

3.  **Code / Terminal (분석적)**
    *   **Font**: *Share Tech Mono, Fira Code, Source Code Pro*.
    *   **Usage**: 장식용 코드, 로그(Log), 힌트, 날짜/시간 표시.
    *   **Style**: `text-xs`나 `text-sm`, 초록색 등 단색 처리.

### 3.3 UI 컴포넌트 가이드
*   **버튼 (Buttons)**:
    *   **형태**: 각진 직사각형(Sharp edges) 또는 아주 살짝 둥근 모서리(`rounded-sm`). 완전한 원형(Pill shape)은 지양(특별한 경우 제외).
    *   **스타일**: 
        *   *Default*: 반투명 배경 + 테두리(`border border-cyan-500/30 bg-cyan-950/20`).
        *   *Hover*: 배경색이 네온처럼 차오르거나 글자색 반전 (`hover:bg-cyan-500 hover:text-black`).
    *   **트랜지션**: 즉각적이고 기계적인 반응 (`duration-150` ~ `200`). 부드러운 페이드보다는 빠릿한 스위칭 선호.
*   **컨테이너 (Panels/Cards)**:
    *   **Glassmorphism**: 배경 블러(`backdrop-blur`)를 사용하여 깊이감 형성.
    *   **Border**: 1px 두께의 얇은 테두리. 빛나는 효과(`shadow-[0_0_15px_...]`)로 강조.

## 4. 인터랙션 & 연출 원칙 (Interaction Principles)

### 4.1 "Sound is Half the Visual" (청각적 피드백 필수)
이 프로젝트에서 소리는 디자인의 일부입니다. 모든 상호작용은 소리를 동반해야 합니다.
*   **Click**: 밋밋한 시스템 소리 금지. `Glass Click`, `Sci-fi Beep`, `Mechanical Switch` 등 상황에 맞는 소리.
*   **Transition**: 화면 전환 시 `Swoosh`, `Glitch noise`, `Data loading` 사운드 활용.
*   **Ambience**: 배경에는 항상 낮은 웅웅거림(`Drone`), 바람 소리, 또는 미세한 전기음이 깔려 있어야 공허하지 않음.

### 4.2 애니메이션 (Motion)
*   **등장 (Enter)**: `Glitch` > `Fade In` > `Slide`. 그냥 나타나지 않고, 데이터가 전송되어 구성되는 느낌(글리치, 디코딩 효과) 선호.
*   **강조 (Emphasize)**: `Pulse` (맥박), `Scanline` (위아래로 훑기).
*   **속도감**: 늘어지는 애니메이션 금지. 빠르고(`0.3s` ~ `0.5s`) 탄력적(`Spring`)이어야 함. 도장 찍기는 `Scale` + `Bounce`로 타격감 부여.

### 4.3 내러티브 연출
*   **대화창 제어**: 사용자가 읽는 속도를 고려하여 텍스트를 한 번에 보여주지 말고, 단계별로 끊어서 보여주거나 타이핑 효과(`Typewriter`) 사용.
*   **시선 유도**: 중요한 오브젝트는 반짝이거나(`Shimmer`) 미세하게 움직여서(`Float`) 사용자가 자연스럽게 클릭하게 유도.

## 5. 테마 변형 가이드 (Expansion Rules)

2탄이나 외전을 만들 때, **'변주 가능한 것'**과 **'지켜야 할 것'**입니다.

### 5.1 반드시 지켜야 할 것 (Fixed Pillars)
1.  **"요원 시점"**: 사용자를 '학생'이나 '유저'가 아닌 **'요원(Agent)'**으로 대우해야 함. (말투: 하십시오체, ~인가?, 확인 바람 등)
2.  **색상 체계의 의미**: `Green=Success`, `Red=Danger`라는 신호등 규칙은 절대 깨지 말 것. (UI 직관성 유지)
3.  **데이터 시각화**: 텍스트를 그냥 보여주지 않고, 항상 프레임, 괄호 `[]`, 장식 선 `//`, `>>` 등을 사용하여 '데이터화'해서 보여줄 것.

### 5.2 자유롭게 변형 가능한 것 (Flexible Elements)
1.  **배경 시대/장소**: 꼭 사이버펑크 도시일 필요 없음. '중세 판타지', '느와르 탐정 사무소', '8비트 게임 속' 등 **"가상 시뮬레이션"**이라는 설정 하에 어떤 테마든 차용 가능. (단, UI 오버레이는 항상 하이테크 스타일 유지 - *대조의 미학*)
2.  **조력자의 모습**: AI, 동물, 유령, 픽셀 캐릭터 등 다양화 가능.
3.  **미니게임 장르**: 퀴즈, 퍼즐, 숨은그림찾기, 슈팅 등 UI/UX 학습에 적합하다면 어떤 방식이든 OK.

## 6. 금지 요소 & 주의사항 (Anti-Patterns)

### 🚫 절대 하지 말아야 할 것
1.  **"귀여움 과잉"**: 둥글둥글하고 파스텔 톤의 유아틱한 디자인 금지. (진지한 요원 훈련 분위기 저해)
2.  **"침묵의 인터랙션"**: 버튼을 눌렀는데 소리도, 시각적 반응도 없는 정적인 상태 금지. 살아있는 시스템 느낌을 줘야 함.
3.  **"지루한 설명충"**: 3줄 이상의 텍스트 덩어리를 한 번에 보여주지 말 것. 대화형으로 쪼개거나 이미지를 활용할 것.
4.  **"시스템 폰트 날것 사용"**: `sans-serif` 기본 폰트가 그대로 노출되면 안 됨. 반드시 지정된 웹폰트가 로드되어야 함.

### ⚠️ 분위기를 깨는 실패 사례
*   배경음악이 갑자기 뚝 끊기는 경우 (Fade out 필수).
*   미래적인 HUD UI 위에 '굴림체'나 '궁서체'가 엉뚱하게 들어가는 경우.
*   경고창(Alert)이 브라우저 기본 팝업(`window.alert`)으로 뜨는 경우. (반드시 커스텀 모달 사용)
